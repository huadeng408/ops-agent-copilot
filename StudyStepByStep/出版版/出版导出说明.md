# 出版导出说明

这份说明用于把 `出版版` 资料稳定地导出成 PDF、打印稿或知识库页面，同时尽量保留 Mermaid 图、章节层级和交叉链接。

## 导出目标

- 打印版：用于纸面复习、批注、讲课。
- PDF 版：用于归档、投递、随身阅读。
- Notion 版：用于拆分成知识库页面，方便打标签和持续补充。

## 推荐导出顺序

1. 先确认课程正文无误，以 `L01-L30` 为主。
2. 再导出索引文件，包括 `README.md`、`课程目录.md`、`参考文献总表.md`、`版本记录.md`。
3. 最后导出 `讲师版脚本` 和 `7天冲刺手册`，作为附录材料。

## PDF 导出建议

### 方案一：VS Code + Markdown PDF

适合快速导出单课文件。

1. 打开任意 `Lxx` 文件。
2. 先用 Markdown Preview 检查 Mermaid 是否正常渲染。
3. 使用 Markdown PDF 或同类扩展导出。
4. 导出后重点检查分页位置、代码链接换行、图表宽度。

### 方案二：Typora / Obsidian

适合导出阅读体验更好的整洁 PDF。

1. 打开 `出版版/README.md` 作为入口。
2. 单课逐个导出，不建议一次性合并 30 节为单个 PDF。
3. 如果需要合刊，建议按主题阶段合并，例如 `L01-L08`、`L09-L18`、`L19-L30`。

### 方案三：Pandoc

适合做正式排版或后续转 `docx`。

1. 先把 Mermaid 图转为静态图或确保目标环境支持 Mermaid。
2. 再使用 Pandoc 统一套用模板、目录和页眉页脚。
3. 如果要投稿或打印装订，建议在 Pandoc 环节统一处理页码和章节编号。

## Notion 导入建议

1. 先创建一级页面 `StudyStepByStep`。
2. 再按 `01-基础认知` 到 `07-面试强化` 建立子页面。
3. `出版版` 里的 `L01-L30` 适合作为数据库条目或独立页面。
4. `讲师版脚本` 建议单独建一个视图，标签设为“口述训练”。

## 导出前检查清单

- [ ] Mermaid 图在预览里能正常显示。
- [ ] 文档中的相对链接没有失效。
- [ ] 中文标题在导出字体下没有乱码。
- [ ] 页脚中的课次、标题、日期一致。
- [ ] “关键源码锚点”和“60秒面试口播模板”没有被分页截断得过碎。

## 建议的归档方式

- `StudyStepByStep/出版版/`：保存 Markdown 源文件。
- `StudyStepByStep/exports/pdf/`：保存导出的 PDF。
- `StudyStepByStep/exports/notion/`：保存迁移说明、截图或导入记录。

## 不建议的做法

- 不建议把 30 节一次性拼接成单个超长 PDF，阅读和后续修订成本都很高。
- 不建议先导出再改源文件，容易出现“PDF 和 Markdown 版本不一致”。
- 不建议删除源码锚点，它们是这套资料和真实工程绑定的核心证据。
