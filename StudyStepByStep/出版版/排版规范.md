# 出版排版规范（v1.0）

## 1. 文档结构规范

每节课统一包含以下块：

1. 课程元信息（标题、课时、受众、先修）
2. 本课定位
3. 图解页（Mermaid）
4. 核心讲解
5. 术语表
6. 面试问题与标准答案
7. 课后任务与参考答案
8. 延伸阅读与参考文献
9. 本课小结

## 2. 版式规范

- 一级标题：`#`
- 二级标题：`##`
- 所有问题采用编号列表，回答使用“答案：”起始。
- 代码路径统一使用绝对路径可点击链接。
- 图解至少 1 张 Mermaid 图。

## 3. 内容深度规范

- 每个问题必须给出“结论 + 原因 + 落地建议”三段式回答。
- 禁止只给定义，不给工程实践场景。
- 优先结合当前仓库源码，不泛化空谈。

## 4. 面试表达规范

- 每课必须可输出 60 秒讲解版本。
- 每课至少 3 个高频追问点。
- 回答顺序建议：结论 -> 证据 -> 取舍 -> 改进。

## 5. 质量检查清单

- 是否包含术语表
- 是否有可执行实战任务
- 是否有标准答案
- 是否有参考文献
- 是否能映射到实际面试问题
